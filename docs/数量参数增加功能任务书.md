# 数量参数增加功能任务书

## 背景
当前工作流在一条指令里默认只生成 1 条草稿；同时“每日新闻”在无提示词时会自动生成 3 条草稿。
这会影响批量生成的可控性，也与“单次生成数量由用户明确指定”的使用预期不一致。

## 目标
- 增加 `--count` 参数，让用户控制单次命令生成的草稿数量（默认 1）。
- 取消“每日新闻无提示词自动生成 3 条”的隐式行为。
- 文档与测试同步更新。

## 范围与改动点
- CLI：`apps.cli create/auto` 新增 `--count`（>=1）。
- 每日新闻：按相关性排序后取前 N 条（N=`--count`，默认 1）。
- 普通标题：同一参数输入下可生成 N 条草稿。
- 文档：README + `docs/工作流新闻任务书.md` + `docs/实现计划书.md`。
- 测试：更新 `pick_news_items` 相关用例。

## 验收标准
- `create/auto --count N` 可生成 N 条草稿；默认 1 条。
- “每日新闻”无提示词时不再自动生成 3 条；数量以 `--count` 为准。
- README 与 docs 描述一致；相关单测通过。
